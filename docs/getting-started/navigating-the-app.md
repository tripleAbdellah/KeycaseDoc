# Navigating the App

KeyCase's interface is organized around a **sidebar navigation** that adapts based on your role and the current context.

## Main Layout

The application is divided into three areas:

- **Sidebar** (left) — Primary navigation with links to all major sections.
- **Content area** (center) — The main workspace where you interact with projects, keywords, flows, and more.
- **Top bar** — Organization selector, user menu, and global actions.

## Sidebar Sections

Depending on your role, you'll see some or all of these navigation items:

| Section | Description |
|---|---|
| **Dashboard** | Overview of recent activity and quick links |
| **Projects** | Browse and manage your projects |
| **Keywords** | View and create keywords within a project |
| **Flows** | Design and organize automation flows |
| **Executions** | Launch and monitor execution runs |
| **Environments** | Manage environment variables |
| **Agents** | Configure and monitor execution agents |
| **Administration** | User management, roles, and organization settings |

## Role-Based Navigation

Not all sections are visible to every user. KeyCase uses role-based access control to show only the sections relevant to your permissions. See [Roles and Permissions](/docs/administration/roles-and-permissions) for details.
