# Glossary

| Term | Definition |
|---|---|
| **Agent** | An execution runtime that connects to the KeyCase server and runs automation flows. |
| **Environment** | A named collection of key-value variables used to configure flows for different targets (e.g., staging, production). |
| **Execution** | A single run of one or more flows, tracked with step-by-step results. |
| **Execution Plan** | A saved configuration specifying which flows to run, with which parameters, on which agent and environment. |
| **Flow** | A sequence of keyword steps that defines an end-to-end automation process. |
| **Instance** (keyword) | A preconfigured version of a keyword with specific parameter values filled in. |
| **Keyword** | A reusable building block representing a single action or test step. |
| **Organization** | The top-level tenant in KeyCase, containing users, projects, and shared resources. |
| **Parameter** | A named input or output value on a keyword. Parameters carry data between steps in a flow. |
| **Project** | A container within an organization that holds keywords, flows, environments, and executions. |
| **Role** | A set of permissions assigned to a user at the organization or project level (e.g., Owner, Admin, Editor, Viewer). |
