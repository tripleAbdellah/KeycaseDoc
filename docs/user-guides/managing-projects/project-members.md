# Project Members

Each project in KeyCase can have multiple members with different levels of access.

## Viewing Members

Navigate to your project's **Settings > Members** to see all current members and their roles.

## Adding Members

1. Open **Settings > Members** within the project.
2. Click **Add Member**.
3. Search for the user by name or email.
4. Assign a role (e.g., Viewer, Editor, Admin).
5. Click **Add** to confirm.

## Changing Roles

To change a member's role, click the role dropdown next to their name in the members list and select the new role.

## Removing Members

Click the remove action next to a member's entry to revoke their access to the project. This does not delete their KeyCase account — it only removes them from the project.

## Role Descriptions

| Role | Permissions |
|---|---|
| **Viewer** | Read-only access to all project resources |
| **Editor** | Create and modify keywords, flows, and executions |
| **Admin** | Full control including member management and settings |

See [Roles and Permissions](/docs/administration/roles-and-permissions) for more details on the permission model.
